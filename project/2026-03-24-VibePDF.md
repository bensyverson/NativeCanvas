# VibePDF Implementation Plan

## Context

VibePDF is a proof-of-concept demo app for the NativeCanvas library, located at `/Users/ben/git/NativeCanvas/Examples/VibePDF/`. It demonstrates an AI-powered document creation workflow: an Operator-powered agent writes JavaScript scripts that render into a NativeCanvas canvas, displayed as a virtual 8.5×11" document. The user interacts via a floating liquid-glass chat interface.

The app is a SwiftUI app targeting **macOS 26+ / iOS 26+ only** (Tahoe/liquid glass era). It's in the NativeCanvas monorepo and already has the `Operator` and `NativeCanvas` packages imported in its Xcode project. It uses **folder-based source management** — new Swift files are auto-discovered, no changes to `project.pbxproj` needed.

**CLAUDE.md constraints that apply:**
- Strict red/green TDD for all logic. Write failing tests first, then implement.
- `@MainActor` is implicit for all declarations in the app module — do NOT add it explicitly to new types.
- No `nonisolated(unsafe)` or `@unchecked Sendable`.
- One type per file. Folders max one level deep.
- `swiftformat . --lint` and `swift test` must pass before committing.
- DocC is not required for the Examples app (only for the NativeCanvas library itself).

## Key Libraries

### NativeCanvas (from `/Users/ben/git/NativeCanvas/Sources/NativeCanvas/`)

**`CanvasRenderer`** — static, nonisolated, all rendering entry points:
```swift
// Renders to CGImage (no scene needed for this app)
CanvasRenderer.render(source: String, viewport: CanvasViewport) throws -> CGImage

// Renders into an existing CGContext (used for PDF export)
CanvasRenderer.render(source: String, into context: CGContext, viewport: CanvasViewport) throws
```

**`CanvasViewport`**:
```swift
CanvasViewport(width: Int, height: Int)  // safeArea defaults to .zero
```

**`CanvasError`** — thrown on bad JS:
- `.evaluationFailed(String)` — JS runtime error message
- `.missingExport(String)` — `schema` or `layers` not exported
- `.invalidLayers` — layers array absent or empty

**Valid NativeCanvas JS template** — must export these two globals:
```javascript
const schema = {
    name: "My Doc",
    params: { /* optional params */ }
};
const layers = [
    {
        name: "Background",
        render: function(ctx, params, scene) {
            // ctx = Canvas 2D API
            // scene.viewport.width / scene.viewport.height for dimensions
            ctx.fillStyle = "#fff";
            ctx.fillRect(0, 0, scene.viewport.width, scene.viewport.height);
        }
    }
];
```
The `nc` standard library is injected automatically: `nc.lerp()`, `nc.rgba()`, `nc.pt()`, `nc.roundRect()`, etc.

### Operator (from `/Users/ben/git/Operator/Sources/Operator/`)

**Tool definition pattern** (from `ClosureTool.swift`):
```swift
try Tool(name: "toolName", description: "...", input: MyInput.self) { input in
    // async closure, return ToolOutput
    return ToolOutput("result")
}
// No-input tool:
Tool(name: "toolName", description: "...") {
    return ToolOutput([.image(data: pngData, mediaType: "image/png", filename: "canvas.png")])
}
```

**ToolInput protocol** (from `ToolInput.swift`): Codable + static `paramDescriptions: [String: String]`.

**Operable protocol** (from `Operable.swift`): Provides `var toolGroup: ToolGroup { get }`.

**ToolGroup** (from `ToolGroup.swift`): `name`, optional `description`, `tools: [any ToolProvider]`.

**Operative creation pattern** (from Chat example `ChatState+Operative.swift`):
```swift
// Cloud providers:
operative = try Operative(name:, description:, provider: llmProvider,
    systemPrompt:, tools:, budget:, configuration:)

// Apple Intelligence (macOS 26+, iOS 26+):
#if canImport(FoundationModels)
if #available(macOS 26.0, iOS 26.0, *) {
    operative = try Operative(name:, description:, systemPrompt:, tools:, budget:)
}
#endif
```

**Streaming pattern** (from Chat example `ChatState+SendMessage.swift`):
```swift
let stream: OperationStream = if let convo = lastConversation {
    operative.run(message, continuing: convo)
} else {
    operative.run(message)
}
for await op in stream {
    switch op {
    case .text(let chunk):   // append to current agent message
    case .toolsRequested:    // show tool calls in chat
    case .toolCompleted:     // show tool output in chat
    case .completed(let r):  lastConversation = r.conversation  // KEY: saves conversation
    case .stopped(let r):    // show stop reason
    default: break
    }
}
```

**Conversation continuity**: `lastConversation` is kept even when `operative` is rebuilt with a new provider. The next `run(_:continuing:)` picks up the history with the new backend.

**Vision support** (from `ModelName+Vision.swift`):
```swift
// Returns true/false/nil
ModelName(rawValue: someString).supportsVision
```

**Provider → LLM.Provider mapping**:
- `.anthropic(apiKey:)` — api.anthropic.com
- `.openAI(apiKey:)` — api.openai.com
- `.mistral(apiKey:)` — api.mistral.ai
- `.lmStudio` — localhost:1234
- `.other(URL, apiKey: String?)` — OpenAI-compatible; use for OpenRouter:
  ```swift
  .other(URL(string: "https://openrouter.ai/api")!, apiKey: key)
  ```
  Pass model name via `ConversationConfiguration(model: ModelName(rawValue: modelName))`

### Agent

**`Agent/CanvasOperable.swift`**

Conforms to `Operable`. Initialised with a reference to `DocumentCoordinator` (captured in tool closures as weak to avoid retain cycle, but guarded — coordinator lives as long as the app).

Three tools in `ToolGroup(name: "Canvas")`:

1. **`write_script`** — `WriteScriptInput: ToolInput` with `script: String`.
   - `await coordinator.setScript(input.script)` — sets `jsScript`, triggers rerender
   - Returns `ToolOutput("Script written successfully.")` or `ToolOutput("Render error: \(error)")`

2. **`edit_script`** — `EditScriptInput: ToolInput` with `old_string: String`, `new_string: String`.
   - Replaces first occurrence; returns error if not found
   - `await coordinator.editScript(old: input.old_string, new: input.new_string)`

3. **`view_canvas`** — no-input tool (use `Tool(name:description:execute:)` factory).
   - Conditionally added to the tools array only when vision is supported
   - Renders current script to `CGImage` → PNG `Data` → returns `ToolOutput([.image(data:, mediaType: "image/png", filename: "canvas.png")])`
   - If no script set: returns `ToolOutput("No script loaded yet.")`

Vision inclusion — in `DocumentCoordinator.buildOperative()`:
```swift
var tools: [any ToolProvider] = [writeScriptTool, editScriptTool]
if settings.provider.supportsVision(modelName: settings.modelName) {
    tools.append(viewCanvasTool)
}
let group = ToolGroup(name: "Canvas", tools: tools)
```

### Coordinator

**`Coordinator/DocumentCoordinator.swift`**

```swift
@Observable final class DocumentCoordinator {
    // State
    var jsScript: String? = nil
    var renderedImage: CGImage? = nil
    var renderError: String? = nil
    var messages: [ChatMessage] = []
    var lastConversation: Conversation? = nil
    var isAgentRunning: Bool = false
    var showHistory: Bool = false
    var showSidebar: Bool = false
    var isExporting: Bool = false
    var exportData: Data? = nil      // PDF data pending save

    var settings: AgentSettings = AgentSettings()
    private var operative: Operative? = nil

    // Computed
    var hasScript: Bool { jsScript != nil }

    // Methods
    func buildOperative()          // creates Operative from settings, keeps lastConversation
    func send(_ text: String) async  // streams agent response
    func setScript(_ script: String) async   // updates jsScript + rerenders
    func editScript(old: String, new: String) async throws  // string replacement + rerender
    func rerender()                // calls CanvasRenderer, updates renderedImage/renderError
    func exportPDF() throws -> Data  // renders to PDF CGContext, returns Data
}
```

**`buildOperative()`** — follows Chat example pattern:
- Does NOT reset `lastConversation`
- Rebuilds `CanvasOperable` with `self` reference and vision-gated tools
- Budget: `Budget(maxTurns: 20)`
- ConversationConfiguration: `model: ModelName(rawValue: settings.modelName)` (nil if empty string)
- Apple Intelligence: `#if canImport(FoundationModels)` + `@available(macOS 26.0, iOS 26.0, *)`

**`rerender()`** — synchronous helper:
```swift
guard let source = jsScript else { renderedImage = nil; return }
do {
    renderedImage = try CanvasRenderer.render(source: source, viewport: settings.viewport)
    renderError = nil
} catch {
    renderError = error.localizedDescription
}
```

**`exportPDF()`** — creates PDF data:
```swift
func exportPDF() throws -> Data {
    guard let source = jsScript else { throw ExportError.noScript }
    let vp = settings.viewport
    var mediaBox = CGRect(x: 0, y: 0, width: vp.width, height: vp.height)
    let data = NSMutableData()
    guard let consumer = CGDataConsumer(data: data),
          let ctx = CGContext(consumer: consumer, mediaBox: &mediaBox, nil)
    else { throw ExportError.contextCreationFailed }
    ctx.beginPDFPage(nil)
    try CanvasRenderer.render(source: source, into: ctx, viewport: vp)
    ctx.endPDFPage()
    ctx.closePDF()
    return data as Data
}
```

**`send(_ text: String)`** — nearly identical to Chat example's `ChatState.send()`:
- Appends user `ChatMessage` immediately
- Streams `OperationStream`, handling `.text`, `.toolsRequested`, `.toolCompleted`, `.toolFailed`, `.completed`, `.stopped`
- Tool calls shown as `ChatMessage(role: .toolCall, text: req.name, toolName: req.name)`
- On `.completed(result)`: `lastConversation = result.conversation`
- New agent text: create message with `isStreaming: true`, append chunks; on `.completed` set `isStreaming: false`

**`setScript` and `editScript`** — both `async` (called from tool closures via `await`):
```swift
func setScript(_ script: String) async {
    jsScript = script
    rerender()
}

func editScript(old: String, new: String) async throws {
    guard var source = jsScript else { throw EditError.noScript }
    guard source.contains(old) else { throw EditError.stringNotFound }
    source.replace(old, with: new)  // first occurrence
    jsScript = source
    rerender()
}
```
Since tool closures run off main-actor and the coordinator is `@MainActor` (implicit), tool code calls `await coordinator.setScript(...)` — Swift's actor isolation handles the dispatch automatically.

## Notes for Implementing Claude

- **@MainActor is implicit** for all app types per CLAUDE.md — never add it explicitly
- **Tool closures & actor isolation**: Since `DocumentCoordinator` is implicitly `@MainActor`, tool closures call `await coordinator.setScript(...)` directly — Swift handles the actor hop
- **OpenRouter** has NO named Provider case — use `.other(URL(string: "https://openrouter.ai/api")!, apiKey: key)`
- **Mistral** has no vision support in our conservative mapping — exclude `view_canvas` for Mistral
- **Apple Intelligence**: `Operable` tools do NOT need `@Generable` inputs for cloud models, but tools with `@Generable` inputs also work on-device. For simplicity, our canvas tools won't have `@Generable` inputs, meaning they'll be silently unavailable with Apple Intelligence (which is acceptable for a demo)
- **`.glassEffect()`** is the iOS 26 / macOS 26 liquid glass SwiftUI modifier — no `@available` guard needed since app is 26-only
- **Xcode auto-discovery**: creating a new .swift file in the `VibePDF/` folder (or subfolder) is enough — Xcode picks it up automatically
- **Chat example uses TextUI** (`@Observed` from TextUI), NOT SwiftUI's `@Observable` — do not copy `@Observed` usage; use `@Observable` instead
- **String replacement**: use `source.replace(old, with: new)` (Swift 5.7+ mutating method) or `source = source.replacingOccurrences(of: old, with: new, options: .literal, range: source.range(of: old))` to replace only first occurrence
- **Rendering profile**: use `.display` (default) for both preview and PDF — HDR not needed here
- **The `fileExporter` binding**: `pdfData.map { PDFExportDocument(data: $0) }` — when nil, fileExporter does nothing; use `Binding<PDFExportDocument?>` pattern
